# 任务执行说明书：把 5 张塔设定图接入游戏

## 【角色定义】

你是一个专门负责把美术设定图接入 Phaser/Vite 塔防游戏的助手。你只做接入，不做切片、不调整数值、不做美术再创作。

## 【任务目标】

把用户提供的 5 张塔设定图（兵营 / 火焰塔 / 宝箱塔 / 魔法塔 / 迫击炮塔）作为静态资源放入仓库，并在游戏 UI 增加一个 “塔图鉴” 入口，玩家点开后能完整查看任意一张设定图。**不修改任何塔的战斗数值，不切片精灵图**。

## 【执行步骤】

1. 在仓库根目录创建文件夹 `public/assets/towers/`（已存在则跳过）。

2. 把用户提供的 5 张 PNG 按下表精确文件名保存到 `public/assets/towers/`。识别依据是图片左上角的中文标题：

   | 标题 | 文件名 |
   |---|---|
   | 兵营 | `barracks_codex.png` |
   | 火焰塔 | `flame_codex.png` |
   | 宝箱塔 | `treasure_codex.png` |
   | 魔法塔 | `mage_codex.png` |
   | 迫击炮塔 | `mortar_codex.png` |

3. 编辑 `src/data/towers.js`：

   - 在 `TOWER_TYPES.barracks`、`TOWER_TYPES.flame`、`TOWER_TYPES.mage`、`TOWER_TYPES.artillery` 这 4 个对象中各新增一个字段（注意是 `artillery` 对应 `迫击炮塔`，不是新增 `mortar` 键）：
     - `barracks` 加 `codexImage: "tower-codex-barracks"`
     - `flame` 加 `codexImage: "tower-codex-flame"`
     - `mage` 加 `codexImage: "tower-codex-mage"`
     - `artillery` 加 `codexImage: "tower-codex-mortar"`
   - 检查 `TOWER_TYPES` 是否有键 `treasure`：
     - 若无：在 `TOWER_TYPES` 对象内末尾追加以下条目（**逐字复制**）：
       ```js
       treasure: {
         name: "宝箱塔",
         price: 90,
         range: 120,
         damage: 12,
         rate: 900,
         projectileSpeed: 380,
         color: 0xc89030,
         accent: 0xfff0a8,
         texture: "tower-artillery",
         projectile: "bomb",
         description: "击杀敌人额外掉落金币",
         blueprintKey: "treasure",
         codexImage: "tower-codex-treasure",
       },
       ```
     - 若有：仅给该条目加上 `codexImage: "tower-codex-treasure"`，不要重复定义其它字段。
   - 如果 `SPECIAL_TOWER_KEYS` 数组里没有字符串 `"treasure"`，把它追加到末尾。
   - 不要把 `treasure` 加入 `TOWER_BUTTON_ORDER`（避免在战斗内出现购买按钮）。

4. 编辑 `src/scenes/GameScene.js`，在类内确保存在 `preload()` 方法：

   - 若类中**已有** `preload()`：在其方法体末尾追加下方 5 行。
   - 若类中**没有** `preload()`：在 `create()` 方法之前新增一个 `preload()` 方法，方法体只放下方 5 行。

   要追加的 5 行：

   ```js
   this.load.image("tower-codex-barracks", "assets/towers/barracks_codex.png");
   this.load.image("tower-codex-flame",    "assets/towers/flame_codex.png");
   this.load.image("tower-codex-treasure", "assets/towers/treasure_codex.png");
   this.load.image("tower-codex-mage",     "assets/towers/mage_codex.png");
   this.load.image("tower-codex-mortar",   "assets/towers/mortar_codex.png");
   ```

   并在 `preload()` 方法体最开头加入一行容错（已存在则不重复加）：

   ```js
   this.load.on("loaderror", () => {});
   ```

5. 仍在 `GameScene.js`，找到 `createUi()` 方法（用文本搜索 `createUi()`）。在该方法体末尾、`updateUi();` 调用之前，追加一段：

   ```js
   this.codexButton = this.createButton(
     PANEL_X + 62, 250, 100, 30, "塔图鉴",
     () => this.openTowerCodexList(),
     { fill: 0x9bc06a, hoverFill: 0xb1d585, stroke: 0x3e6a26, color: "#1f3013" }
   );
   ```

   坐标 `PANEL_X` 已从 `../data/map.js` 导入；如未导入则在文件顶部 import 列表中已有 `PANEL_X` 时不要重复添加。

6. 在 `GameScene` 类中新增两个方法（放在 `createUi` 之后任意位置）：

   ```js
   openTowerCodexList() {
     this.createModalBase("塔图鉴", "选择一座塔查看完整设定图。");
     const order = ["barracks", "flame", "treasure", "mage", "artillery"];
     order.forEach((key, index) => {
       const type = TOWER_TYPES[key];
       if (!type) return;
       const col = index % 3;
       const row = Math.floor(index / 3);
       const x = 200 + col * 200;
       const y = 200 + row * 60;
       this.createButton(x, y, 170, 40, type.name, () => {
         this.closeModal();
         this.openTowerCodex(key);
       }, { fill: 0xe7c980, hoverFill: 0xf0d894, stroke: 0x8a5a26, color: "#3a2816" });
     });
     this.addModalCloseButton("关闭", () => this.closeModal());
   }

   openTowerCodex(typeKey) {
     const type = TOWER_TYPES[typeKey];
     if (!type || !type.codexImage || !this.textures.exists(type.codexImage)) {
       this.showNotice(`暂无 ${type?.name ?? typeKey} 图鉴`, "#7a4b25");
       return;
     }
     this.createModalBase(`${type.name} · 图鉴`, "点击关闭返回。");
     const image = this.add.image(GAME_WIDTH / 2, GAME_HEIGHT / 2 + 16, type.codexImage).setDepth(81);
     const maxW = GAME_WIDTH - 80;
     const maxH = GAME_HEIGHT - 140;
     const scale = Math.min(maxW / image.width, maxH / image.height, 1);
     image.setScale(scale);
     this.modalObjects.push(image);
     this.addModalCloseButton("关闭", () => this.closeModal());
   }
   ```

   `createModalBase`、`modalObjects`、`addModalCloseButton`、`closeModal`、`showNotice`、`createButton` 必须为类已有方法 —— 不要重新实现。`TOWER_TYPES`、`GAME_WIDTH`、`GAME_HEIGHT`、`PANEL_X` 必须已在文件顶部 import；若缺失则补 import，不要在方法内 import。

7. 运行：

   ```powershell
   npm run build
   ```

   退出码必须为 0，且终端无 `error` 字样。否则回滚步骤 3–6 的修改并报告错误。

8. 运行：

   ```powershell
   npm run dev -- --port 5173
   ```

   浏览器打开后：点击右侧面板新增的 “塔图鉴” → 弹出 5 个塔名按钮 → 依次点击每个按钮，确认 5 张原图都能完整显示且能通过 “关闭” 返回。

## 【输入说明】

- **图片来源**：用户在对话中以 5 张 PNG 附件提供，每张约 1024×683 像素，左上角带中文标题。
- **代码仓库**：根目录 `c:\MyGame`，主版本在 `src/`。重点文件：
  - `src/data/towers.js`
  - `src/scenes/GameScene.js`
  - `src/data/map.js`（提供 `GAME_WIDTH`、`GAME_HEIGHT`、`PANEL_X`）
- **设计参考**：`doc/game-design.md` §12 已列出特殊塔；本任务不修改其中任何数值。

输入示例：第 2 张图标题为 “火焰塔”，下方 7 个塔形态从 “等级1” 到 “等级4（灼烧型）” → 必须保存为 `flame_codex.png`。

## 【输出要求】

必须包含：

- `public/assets/towers/` 下 5 个 PNG，命名严格匹配第 2 步表格。
- `src/data/towers.js` 中 5 个塔键（barracks / flame / mage / artillery / treasure）各带 `codexImage` 字段；`SPECIAL_TOWER_KEYS` 含 `"treasure"`。
- `src/scenes/GameScene.js` 中存在 `preload()` 含 5 行加载语句、`createUi` 末尾的 “塔图鉴” 按钮、新方法 `openTowerCodexList`、`openTowerCodex`。
- `npm run build` 通过。

明确禁止：

- 不要修改任何塔的 `damage` / `rate` / `range` / `price` / `splash` / `slowMs` 等数值字段。
- 不要切片、裁剪、二次绘制原图，整图展示。
- 不要把 `treasure` 加入 `TOWER_BUTTON_ORDER`。
- 不要新增任何 npm 依赖。
- 不要修改 `project.godot`、根目录 `scenes/`、`scripts/`、`docs/GODOT_MIGRATION.md`。
- 不要修改 `doc/game-design.md`。
- 不要在 PowerShell 里用 `Get-Content | Set-Content` 处理含中文的源码或 md 文件。

## 【边界与限制】

- 如果 5 张图中任意一张缺失或左上角标题与表格不符：跳过该张图的所有相关步骤，控制台输出 `[待确认] 缺失或无法识别的塔图：<期望文件名>`，继续处理其它图。
- 如果 `GameScene.js` 没有 `createButton` / `createModalBase` / `closeModal` / `showNotice` 中任意一个：**停止执行**，输出 `[待确认] GameScene 缺少基础 UI 方法：<方法名>`，并说明无法继续接入图鉴 UI 的原因。
- 如果 `TOWER_TYPES` 中已经有 `treasure` 键且字段冲突（例如已有 `codexImage` 指向不同 key）：保留原值，**不要覆盖**，控制台输出 `[待确认] treasure.codexImage 已存在，跳过覆盖`。
- 如果 `npm run build` 因 import 缺失失败：先检查 `GameScene.js` 顶部是否同时 import 了 `GAME_WIDTH`、`GAME_HEIGHT`、`PANEL_X`、`TOWER_TYPES`，缺哪个补哪个；仍不通过则回滚并报错。
- 如果用户后续要求 “拆分精灵图” 或 “逐级显示塔外观”：**本任务不做**，回复 `[待确认] 拆分精灵需新建任务说明书`。

## 【示例】

正例：

- 用户第 1 张图标题 “兵营” → 保存 `public/assets/towers/barracks_codex.png` → `TOWER_TYPES.barracks.codexImage = "tower-codex-barracks"` → 浏览器点 “塔图鉴 → 卫兵营” 完整显示设定图 → `npm run build` 退出码 0。
- 用户第 5 张图标题 “迫击炮塔” → 保存 `mortar_codex.png` → 在 **现有键 `artillery`** 上加 `codexImage: "tower-codex-mortar"` → 点 “塔图鉴 → 矮炮台” 显示该图。

反例：

- 把 “兵营” 图存到 `public/barracks.png`（错误：路径不在 `public/assets/towers/`、命名缺 `_codex`）。
- 看到第 5 张标题 “迫击炮塔” 就在 `TOWER_TYPES` 里新增 `mortar` 键（错误：应复用现有 `artillery`）。
- 把 `TOWER_TYPES.barracks.damage` 由 `8` 改成 `10`（错误：禁止改数值）。
- 把宝箱塔图替换掉现有 `tower-arrow` 纹理（错误：本任务不替换战斗内塔纹理）。
- 把 `treasure` 追加到 `TOWER_BUTTON_ORDER` 让它在战斗里能买（错误：本任务只做图鉴）。
- 在 `preload` 之外、`create` 里调用 `this.load.image(...)`（错误：Phaser 资源必须在 `preload` 加载）。
