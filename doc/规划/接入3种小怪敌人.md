# 任务执行说明书：把绿史莱姆 / 电猫 / 石巨人 接入为游戏敌人

## 【角色定义】

你是一个专门负责把美术敌人精灵图接入 Phaser/Vite 塔防游戏的助手。你只做接入，不做切片、不调整数值、不修改动画时长以外的战斗参数。

## 【任务目标】

把用户提供的 1 张敌人合集图（含绿史莱姆 / 电猫 / 石巨人，共 3 种敌人 × 4 状态 [Walk/Run/Attack/Dead] × 3 帧）作为静态资源接入仓库，并完成两件事：

1. **图鉴接入（必做）**：仿照已有 “塔图鉴”，新增 “敌人图鉴” 按钮，点开后可完整查看这张原图。
2. **战斗外观替换（条件做）**：当用户额外提供了已切好的精灵条文件时，把游戏中 3 个 rank（normal / rare / heavy）的纹理替换为对应敌人的 walk 行走动画。**不要自行从合集图里抠图**。

数值、波次、掉落概率、Boss 设定 **完全不动**。

## 【执行步骤】

### A 阶段：图鉴接入（无条件执行）

1. 创建文件夹 `public/assets/enemies/`（已存在则跳过）。

2. 把用户提供的合集图保存为 `public/assets/enemies/enemy_codex.png`。识别依据：图片顶部并列写着 `Green Slime / 绿史莱姆`、`Volt Kitten / 电猫`、`Rock Golem / 石巨人`。

3. 编辑 `src/scenes/GameScene.js`：

   - 在已有的 `preload()` 方法体末尾追加（若类无 `preload()` 则按 `doc/规划/接入5张塔图鉴.md` 第 4 步建立 `preload()` 后再追加；并在该方法体最开头确保有一行 `this.load.on("loaderror", () => {});`）：

     ```js
     this.load.image("enemy-codex", "assets/enemies/enemy_codex.png");
     ```

4. 在 `GameScene` 类中新增方法 `openEnemyCodex()`：

   ```js
   openEnemyCodex() {
     if (!this.textures.exists("enemy-codex")) {
       this.showNotice("暂无敌人图鉴", "#7a4b25");
       return;
     }
     this.createModalBase("敌人图鉴", "点击关闭返回。");
     const image = this.add.image(GAME_WIDTH / 2, GAME_HEIGHT / 2 + 16, "enemy-codex").setDepth(81);
     const maxW = GAME_WIDTH - 80;
     const maxH = GAME_HEIGHT - 140;
     const scale = Math.min(maxW / image.width, maxH / image.height, 1);
     image.setScale(scale);
     this.modalObjects.push(image);
     this.addModalCloseButton("关闭", () => this.closeModal());
   }
   ```

5. 在 `createUi()` 方法体末尾、`updateUi();` 之前，且**紧接** “塔图鉴” 按钮之后，追加一个按钮：

   ```js
   this.enemyCodexButton = this.createButton(
     PANEL_X + 62, 286, 100, 30, "敌人图鉴",
     () => this.openEnemyCodex(),
     { fill: 0xc9756d, hoverFill: 0xdd8d85, stroke: 0x5c2a25, color: "#fff6d8" }
   );
   ```

   （Y 坐标 286 = “塔图鉴” 的 250 + 36 间距；若 “塔图鉴” 按钮不存在则**停止本步骤**，控制台输出 `[待确认] 缺少塔图鉴按钮，先执行 doc/规划/接入5张塔图鉴.md` 后跳过本步。）

6. 运行 `npm run build`，必须退出码 0。

### B 阶段：战斗外观替换（仅当用户已提供切好的精灵条时执行）

7. 检查 `public/assets/enemies/` 目录里是否同时存在以下 3 个文件：

   | 文件 | 内容 | 帧规格 |
   |---|---|---|
   | `slime_walk.png` | 绿史莱姆行走 3 帧水平排列 | 3 帧等宽，单帧正方形 |
   | `kitten_walk.png` | 电猫行走 3 帧水平排列 | 同上 |
   | `golem_walk.png`  | 石巨人行走 3 帧水平排列 | 同上 |

   若 **任一文件缺失**：跳过 B 阶段全部步骤，控制台输出 `[待确认] 缺少切好的敌人行走精灵条：<缺失文件名>`，结束任务。

8. 在 `preload()` 末尾追加（每个文件独立加载为 spritesheet，`frameWidth` 取 `图片宽度 / 3`、`frameHeight` 取 `图片高度`，必须四舍五入到整数；不知道宽高就读取后再算 —— 见步骤 9 的兜底）：

   ```js
   this.load.spritesheet("enemy-slime-walk",  "assets/enemies/slime_walk.png",  { frameWidth: 64, frameHeight: 64 });
   this.load.spritesheet("enemy-kitten-walk", "assets/enemies/kitten_walk.png", { frameWidth: 64, frameHeight: 64 });
   this.load.spritesheet("enemy-golem-walk",  "assets/enemies/golem_walk.png",  { frameWidth: 96, frameHeight: 96 });
   ```

9. 在 `create()` 方法**最开头**（`super.create?.()` 或第一行业务逻辑之前）追加：

   ```js
   ["slime", "kitten", "golem"].forEach((name) => {
     const key = `enemy-${name}-walk`;
     if (!this.textures.exists(key)) return;
     if (this.anims.exists(`${name}-walk`)) return;
     this.anims.create({
       key: `${name}-walk`,
       frames: this.anims.generateFrameNumbers(key, { start: 0, end: 2 }),
       frameRate: 6,
       repeat: -1,
     });
   });
   ```

10. 修改 `spawnEnemy(rank)` 方法。**仅修改 `rankStats` 表中的 `texture` 字段** 与 `tint` 字段，不要改其它字段：

    | rank | 旧 texture | 新 texture | 旧 tint | 新 tint |
    |---|---|---|---|---|
    | `normal` | `enemy-scout` | `enemy-slime-walk` | `0xffffff` | `0xffffff` |
    | `rare`   | `enemy-scout` | `enemy-kitten-walk` | `0xb8deff` | `0xffffff` |
    | `heavy`  | `enemy-brute` | `enemy-golem-walk` | `0xffffff` | `0xffffff` |
    | `elite`  | `enemy-brute` | `enemy-golem-walk` | `0xd6bbff` | `0xd6bbff` |
    | `boss`   | `enemy-brute` | `enemy-golem-walk` | `0xffcf70` | `0xffcf70` |

11. 在 `spawnEnemy` 创建敌人精灵 `this.add.image(...)` 的位置：把 `this.add.image` 改为 `this.add.sprite`（仅当 `rankStats[rank].texture` 以 `enemy-` 起头且对应动画存在时），并在 sprite 创建后立刻调用 `sprite.play(\`${name}-walk\`)`，其中 `name` 为 `slime` / `kitten` / `golem`，由当前 `rank` 决定（normal→slime, rare→kitten, heavy/elite/boss→golem）。

12. 重新运行 `npm run build`，必须退出码 0。

13. 运行 `npm run dev -- --port 5173`，进入战斗，确认：
    - 普通小怪显示为绿史莱姆，三帧循环。
    - 稀有怪显示为黄色电猫。
    - 重型 / 精英 / Boss 显示为石巨人，elite 偏紫色调，boss 偏金色调，体型缩放沿用原有 `scale` 字段。

## 【输入说明】

- **必有图片**：用户在对话中以 PNG 附件提供 1 张约 1024×683 的合集图，包含 3 列 × 4 行 = 12 个动作单元，每单元 3 帧。
- **可选图片**：用户后续可能提供 3 个已切好的水平 3 帧精灵条 `slime_walk.png` / `kitten_walk.png` / `golem_walk.png`；其它动作（Run / Attack / Dead）的精灵条本任务**不接入**。
- **代码仓库**：根 `c:\MyGame`，主版本 `src/`。重点文件：
  - `src/scenes/GameScene.js`（含 `preload`、`createUi`、`spawnEnemy`、`createModalBase` 等已有方法）。
  - `src/data/map.js` 提供 `GAME_WIDTH` / `GAME_HEIGHT` / `PANEL_X`。
- **设计参考**：`doc/game-design.md` §10 敌人掉落、§5.3 18 波节奏；本任务不修改其中任何数值。

输入示例：合集图左上角第一格三只绿色小球 → 必须保存为 `enemy_codex.png`（仅整图保存，不切）。

## 【输出要求】

A 阶段必须包含：

- `public/assets/enemies/enemy_codex.png` 存在。
- `GameScene.js` 中 `preload` 含 `this.load.image("enemy-codex", ...)`。
- `GameScene.js` 中存在 `openEnemyCodex()` 方法。
- `createUi` 末尾创建了 “敌人图鉴” 按钮，与 “塔图鉴” 按钮在 X 同列、Y 间距 36 像素。
- `npm run build` 通过。

B 阶段（仅当 3 个 walk 精灵条都存在时）必须包含：

- `preload` 含 3 行 `this.load.spritesheet(...)`。
- `create` 开头注册了 `slime-walk` / `kitten-walk` / `golem-walk` 3 个动画。
- `spawnEnemy` 中 5 个 rank 的 `texture` 字段已按上方表格替换；rare 的 `tint` 改为 `0xffffff`。
- 战斗中能看到 3 种敌人各自的循环动画。

明确禁止：

- 不要修改任何 rank 的 `hp` / `hpGrowth` / `reward` / `rewardGrowth` / `speed` / `scale` / `damage`。
- 不要从合集图 `enemy_codex.png` 自动抠图、自动切帧、自动用 canvas 切图。
- 不要在 `data` 目录新增 `enemies.js` 文件（本任务不抽数据层）。
- 不要新增 npm 依赖。
- 不要修改 `project.godot`、根目录 `scenes/`、`scripts/`、`docs/GODOT_MIGRATION.md`、`doc/game-design.md`。
- 不要改用其它 frameRate（必须为 6）或其它循环模式（必须为 `repeat: -1`）。
- 不要在 PowerShell 用 `Get-Content | Set-Content` 处理含中文的源码或 md 文件。

## 【边界与限制】

- 如果 `GameScene.js` 不存在 `preload()`：按 `doc/规划/接入5张塔图鉴.md` 第 4 步先建立 `preload()`，再继续。
- 如果 `createUi()` 中找不到 “塔图鉴” 按钮：A 阶段第 5 步控制台输出 `[待确认] 缺少塔图鉴按钮，先执行 doc/规划/接入5张塔图鉴.md`，跳过该步，但仍执行 1–4、6 步。
- 如果 B 阶段中 `frameWidth` / `frameHeight` 推断错误（运行后帧位错位）：先用纸笔测量精灵条的实际像素，按 `图片像素宽 / 3` 取整作为 `frameWidth`、整图高度作为 `frameHeight`，**不要**靠目测调整其它字段。无法测量时控制台输出 `[待确认] slime_walk.png 帧尺寸未知` 并跳过 B 阶段。
- 如果 `spawnEnemy` 的实现里没有 `rankStats` 这个本地变量名（被重构过）：停止 B 阶段执行，输出 `[待确认] spawnEnemy 结构与说明书不一致`，并报告实际函数体首 20 行。
- 如果发现 `boss` 关需要不同模型（例如石巨人不够分量）：本任务**不替换 boss 模型**，沿用石巨人 + 金色 tint + 原 scale。后续若用户提出 “Boss 用专属图”，回复 `[待确认] 需新建 Boss 模型任务说明书`。
- 不确定动画播放调用应放在哪一行时：放在 sprite 创建之后、`enemy.sprite = sprite` 赋值之前。

## 【示例】

正例：

- 用户提供 1 张合集图 → 保存 `public/assets/enemies/enemy_codex.png` → A 阶段完成 → 浏览器点 “敌人图鉴” 看到完整原图 → `npm run build` 退出码 0。
- 用户随后又提供 `slime_walk.png` (192×64) / `kitten_walk.png` (192×64) / `golem_walk.png` (288×96) → B 阶段执行 → 普通波出现行走的绿史莱姆动画。

反例：

- 用 canvas 把合集图裁成 12 张子图（错误：明令禁止抠图）。
- 把 normal 的 `hp: 50` 改成 `60` 来 “适配新外观”（错误：禁止改数值）。
- 在 `boss` 上把 texture 改成专门的 `enemy-boss-walk`（错误：本任务沿用石巨人）。
- 把动画 frameRate 设为 12（错误：必须 6）。
- 给 “敌人图鉴” 起按钮文字 “怪物列表”（错误：必须文字 “敌人图鉴”）。
- 在 `preload` 里调用 `this.add.image(...)`（错误：`add.image` 必须在 `create` 之后）。
