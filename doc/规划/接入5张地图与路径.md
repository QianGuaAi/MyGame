# 任务执行说明书：用 5 张设定图替换游戏地图、重做路径与塔位

## 【角色定义】

你是一个专门负责把章节地图设定图接入 Phaser/Vite 塔防游戏的助手。你只做接入与数据迁移，**不做美术再创作、不修改战斗数值、不改怪物波次**。

## 【任务目标】

1. 把用户提供的 5 张地图设定图作为 5 个章节的战斗背景图。
2. **删除** 原有硬编码的 `PATH_POINTS` 与 `TOWER_SLOTS`（单一全局路径与塔位）。
3. **新增** 按章节区分的路径与塔位数据 `CHAPTER_LAYOUT[0..4]`，每章一组路径折线 + 一组塔位坐标。
4. 战斗内：怪物沿当前章节的路径走，玩家只能在当前章节的塔位上建塔；原本程序化绘制的草地路径与圆形塔位平台 **不再绘制**（被背景图代替）。
5. 提供一个调试模式：按 `D` 键开启时，鼠标点击地图会在控制台打印 `{ x, y }`，便于操作者在真实背景图上手工标定路径与塔位坐标。
6. 数值、波次、英雄设定、Boss 行为 **完全不动**。

   - **新增** 顶层导出函数：

     ```js
     export function getChapterPathSegments(chapterIndex) {
       return buildPathSegments(CHAPTER_LAYOUT[chapterIndex].pathPoints);
     }

     export function getChapterPathLength(chapterIndex) {
       const segs = getChapterPathSegments(chapterIndex);
       return segs[segs.length - 1].end;
     }

     export function pointOnChapterPath(chapterIndex, distance) {
       const segs = getChapterPathSegments(chapterIndex);
       const total = segs[segs.length - 1].end;
       const clamped = Math.max(0, Math.min(distance, total));
       const seg = segs.find((s) => clamped <= s.end) || segs[segs.length - 1];
       const t = seg.length === 0 ? 0 : (clamped - seg.start) / seg.length;
       return {
         x: seg.from.x + (seg.to.x - seg.from.x) * t,
         y: seg.from.y + (seg.to.y - seg.from.y) * t,
         angle: seg.angle,
       };
     }
     ```

### D 阶段：调试工具（坐标拾取）

7. 在 `GameScene` 类中新增方法 `enableMapCalibrationMode()`：

   ```js
   enableMapCalibrationMode() {
     if (this.calibrationEnabled) return;
     this.calibrationEnabled = true;
     this.input.on("pointerdown", (p) => {
       const x = Math.round(p.worldX);
       const y = Math.round(p.worldY);
       console.log(`[map] ch${this.chapterIndex + 1} click: [${x}, ${y}]`);

     this.showNotice("已开启地图标定模式：点击空白记录坐标", "#2f4972");
   ```

