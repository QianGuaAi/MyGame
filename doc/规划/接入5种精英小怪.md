# 任务执行说明书：把 5 种精英小怪接入游戏（按章节归属）

## 【角色定义】

你是一个专门负责把美术精英小怪精灵图接入 Phaser/Vite 塔防游戏的助手。你只做接入与外观替换，**不做美术再创作、不修改 elite rank 的 hp / speed / damage / scale / reward 数值、不修改波次结构、不切片合集图**。

## 【任务目标】

把用户提供的 1 张精英小怪合集图（含 5 种精英：绿史莱姆 / 电猫 / 岩石像 / 蘑菇精灵 / 南瓜鬼）接入游戏，做两件事：

1. **图鉴接入（必做）**：扩展现有 “敌人图鉴”（若不存在则按 `doc/规划/接入3种小怪敌人.md` 流程先建立），新增一个二级入口 “精英小怪” 显示这张合集原图。
2. **外观替换（条件做）**：当用户额外提供 5 个已切好的 “行走 3 帧水平精灵条” 时，把 5 个章节各自的 `elite` rank 敌人外观替换为对应章节的精英小怪。**不要自行从合集图里抠图**。

5 种精英按下表归属章节（依据：本作 `doc/game-design.md` §5.2 各章敌人风格 + 美术主色调匹配）：

| 章节 | 精英小怪 | 文件名前缀 | 匹配理由 |
|---:|---|---|---|
| 第 1 章 边境平原 | 绿史莱姆精英 | `slime_elite` | 平原低阶杂兵自然进阶 |
| 第 2 章 幽暗森林 | 蘑菇精灵 | `mushroom_elite` | 森林蘑菇生态 |
| 第 3 章 烈焰火山 | 南瓜鬼 | `pumpkin_elite` | 火焰 / 灼烧主题 |
| 第 4 章 极北雪山 | 岩石像 | `statue_elite` | 冰封要塞石像守卫 |
| 第 5 章 回响之渊 | 电猫精英 | `kitten_elite` | 奥术 / 电荷主题 |

## 【执行步骤】

### A 阶段：图鉴接入（无条件执行）

1. 在 `public/assets/enemies/` 下创建子目录 `elite/`（若 `enemies/` 不存在先建）。

2. 把用户在本次对话提供的合集 PNG 保存为 `public/assets/enemies/elite/elite_codex.png`。识别依据：图片左上 / 顶部并列写着 `绿史莱姆 / 电猫 / 岩石像 / 蘑菇精灵 / 南瓜鬼` 五列。

3. 编辑 `src/scenes/GameScene.js`：

   - 在 `preload()` 末尾追加：

     ```js
     this.load.image("enemy-elite-codex", "assets/enemies/elite/elite_codex.png");
     ```

   - 找到 `openEnemyCodex()` 方法（若不存在则按 `doc/规划/接入3种小怪敌人.md` 第 4 步先建立）。把它从 “直接显示一张图” 改造为 “2 个分类按钮”：

     ```js
     openEnemyCodex() {
       this.createModalBase("敌人图鉴", "选择一类查看完整设定图。");
       this.createButton(280, 200, 200, 44, "普通小怪", () => {
         this.closeModal();
         this.showCodexImage("enemy-codex", "敌人图鉴 · 普通小怪");
       }, { fill: 0xc9756d, hoverFill: 0xdd8d85, stroke: 0x5c2a25, color: "#fff6d8" });
       this.createButton(280, 260, 200, 44, "精英小怪", () => {
         this.closeModal();
         this.showCodexImage("enemy-elite-codex", "敌人图鉴 · 精英小怪");
       }, { fill: 0x9c5cc4, hoverFill: 0xb074d8, stroke: 0x4a285c, color: "#fff6d8" });
       this.addModalCloseButton("关闭", () => this.closeModal());
     }

     showCodexImage(textureKey, title) {
       if (!this.textures.exists(textureKey)) {
         this.showNotice(`暂无图鉴：${title}`, "#7a4b25");
         return;
       }
       this.createModalBase(title, "点击关闭返回。");
       const image = this.add.image(GAME_WIDTH / 2, GAME_HEIGHT / 2 + 16, textureKey).setDepth(81);
       const maxW = GAME_WIDTH - 80, maxH = GAME_HEIGHT - 140;
       const scale = Math.min(maxW / image.width, maxH / image.height, 1);
       image.setScale(scale);
       this.modalObjects.push(image);
       this.addModalCloseButton("关闭", () => this.closeModal());
     }
     ```

   - 若 `openEnemyCodex` 已经在弹模态后直接 `this.add.image("enemy-codex", ...)`，把那段直接画图的代码替换为上述结构；不要保留两套并行实现。

### B 阶段：精英外观替换（仅当用户已提供切好的精灵条时执行）

4. 检查 `public/assets/enemies/elite/` 下是否同时存在以下 5 个文件（每个都是 “行走 3 帧水平排列” 的精灵条，3 帧等宽，单帧正方形）：

   | 文件 | 对应章节 | 精灵条键名 |
   |---|---:|---|
   | `slime_elite_walk.png` | 1 | `enemy-elite-slime-walk` |
   | `mushroom_elite_walk.png` | 2 | `enemy-elite-mushroom-walk` |
   | `pumpkin_elite_walk.png` | 3 | `enemy-elite-pumpkin-walk` |
   | `statue_elite_walk.png` | 4 | `enemy-elite-statue-walk` |
   | `kitten_elite_walk.png` | 5 | `enemy-elite-kitten-walk` |

   若 **任一文件缺失**：跳过 B 阶段全部步骤，控制台输出 `[待确认] 缺少切好的精英行走精灵条：<缺失文件名>`，结束任务（A 阶段已交付即视为部分完成）。

5. `preload()` 末尾追加 5 行（`frameWidth` 取 `图片宽度 / 3` 取整、`frameHeight` 取整图高度，先用估算值，操作者实测后调整）：

   ```js
   this.load.spritesheet("enemy-elite-slime-walk",    "assets/enemies/elite/slime_elite_walk.png",    { frameWidth: 96, frameHeight: 96 });
   this.load.spritesheet("enemy-elite-mushroom-walk", "assets/enemies/elite/mushroom_elite_walk.png", { frameWidth: 96, frameHeight: 96 });
   this.load.spritesheet("enemy-elite-pumpkin-walk",  "assets/enemies/elite/pumpkin_elite_walk.png",  { frameWidth: 96, frameHeight: 96 });
   this.load.spritesheet("enemy-elite-statue-walk",   "assets/enemies/elite/statue_elite_walk.png",   { frameWidth: 96, frameHeight: 96 });
   this.load.spritesheet("enemy-elite-kitten-walk",   "assets/enemies/elite/kitten_elite_walk.png",   { frameWidth: 96, frameHeight: 96 });
   ```

6. 在 `create()` 注册动画的循环（若已存在 `["slime","kitten","golem"].forEach(...)`）后追加：

   ```js
   const eliteList = [
     ["elite-slime", "enemy-elite-slime-walk"],
     ["elite-mushroom", "enemy-elite-mushroom-walk"],
     ["elite-pumpkin", "enemy-elite-pumpkin-walk"],
     ["elite-statue", "enemy-elite-statue-walk"],
     ["elite-kitten", "enemy-elite-kitten-walk"],
   ];
   eliteList.forEach(([name, key]) => {
     if (!this.textures.exists(key)) return;
     if (this.anims.exists(`${name}-walk`)) return;
     this.anims.create({
       key: `${name}-walk`,
       frames: this.anims.generateFrameNumbers(key, { start: 0, end: 2 }),
       frameRate: 6,
       repeat: -1,
     });
   });
   ```

   `frameRate` 必须 6、`repeat` 必须 -1，与已有 walk 动画保持一致。

7. 在 `src/data/map.js` 的 `CHAPTER_LAYOUT` 每项中**新增** `eliteTexture` 字段（若 `CHAPTER_LAYOUT` 不存在，按 `doc/规划/接入5张地图与路径.md` 重构后再来）。把对应键名填入：

   | 下标 | `eliteTexture` |
   |---:|---|
   | 0 | `"enemy-elite-slime-walk"` |
   | 1 | `"enemy-elite-mushroom-walk"` |
   | 2 | `"enemy-elite-pumpkin-walk"` |
   | 3 | `"enemy-elite-statue-walk"` |
   | 4 | `"enemy-elite-kitten-walk"` |

   每行末加注释 `// [待确认] 章节-精英映射，可由后续任务交换`。

8. 修改 `GameScene.spawnEnemy(rank)`：

   - 在 `rank === "elite"` 分支前添加：

     ```js
     let eliteKey = null;
     let eliteAnim = null;
     if (rank === "elite") {
       const layout = this.layout;
       if (layout?.eliteTexture && this.textures.exists(layout.eliteTexture)) {
         eliteKey = layout.eliteTexture;
         const m = layout.eliteTexture.match(/^enemy-(elite-[a-z]+)-walk$/);
         eliteAnim = m ? `${m[1]}-walk` : null;
       }
     }
     ```

   - 在创建敌人精灵的位置：当 `rank === "elite"` 且 `eliteKey` 不为 null：用 `this.add.sprite(point.x, point.y, eliteKey)` 替换原 `this.add.image(point.x, point.y, "enemy-brute")`；并在 sprite 创建后立即 `if (eliteAnim) sprite.play(eliteAnim);`。
   - 同时把 `rank === "elite"` 的 `tint` 由 `0xd6bbff` 改为 `0xffffff`（避免破坏新美术色调）。**仅** elite 改 tint，其它 rank 不动。
   - `hp / hpGrowth / reward / rewardGrowth / speed / scale / damage` 字段 **完全不动**。

9. 运行 `npm run build`，退出码必须 0。

### 验收

10. 运行 `npm run dev -- --port 5173`：

    - 点击右侧面板 “敌人图鉴” → 弹出 2 个分类按钮 → 点 “精英小怪” → 显示完整合集原图。
    - 5 章战斗中，elite 波（默认在第 9、第 13 波出现）出现的精英外观与本说明书表格一致：
      - 第 1 章 → 绿色精英史莱姆
      - 第 2 章 → 蘑菇精灵
      - 第 3 章 → 南瓜鬼
      - 第 4 章 → 岩石像
      - 第 5 章 → 电猫精英
    - elite 走路动画为 3 帧循环；其它 rank 外观不变。

## 【输入说明】

- **必有图片**：用户在对话中以 1 张约 1024×580 的合集 PNG 提供，含 5 列纵向面板。
- **可选图片**：用户后续可能提供 5 个 `*_elite_walk.png` 行走精灵条（水平 3 帧）。
- **代码仓库**：根 `c:\MyGame`，主版本 `src/`。
- **重点文件**：`src/data/map.js`、`src/scenes/GameScene.js`。
- **设计参考**：`doc/game-design.md` §5.2、§10。

输入示例：合集图最左列写 “绿史莱姆” → 必须保存为 `elite_codex.png`（仅整图）；若用户后续提供 `slime_elite_walk.png`（约 288×96，3 帧），第 1 章 elite 替换生效。

## 【输出要求】

A 阶段必须包含：

- `public/assets/enemies/elite/elite_codex.png` 存在。
- `preload` 含 `this.load.image("enemy-elite-codex", ...)`。
- `openEnemyCodex` 改造为分类入口；新增 `showCodexImage` 方法。
- 点击 “敌人图鉴 → 精英小怪” 能完整显示合集图。

B 阶段（仅当 5 个 walk 精灵条全在时）必须包含：

- `preload` 含 5 行 `loadSpritesheet` 加载语句。
- `create` 注册 5 个 `elite-*-walk` 动画。
- `CHAPTER_LAYOUT[0..4]` 各含 `eliteTexture` 字段，值与表格匹配。
- `spawnEnemy` 的 `elite` 分支按 `layout.eliteTexture` 选纹理并播放对应动画；elite tint 改为 `0xffffff`。
- 战斗中 5 章 elite 外观分别正确。

明确禁止：

- 不要修改 `getCurrentRank` 中 elite 的出现波次 / 概率。
- 不要修改 `spawnEnemy` 中 elite 的 `hp/hpGrowth/reward/rewardGrowth/speed/scale/damage`。
- 不要修改其它 rank（normal / rare / heavy / boss）的纹理或 tint。
- 不要从合集图自动抠图、不要在 canvas 里二次切片。
- 不要新增 npm 依赖。
- 不要修改 `project.godot`、根 `scenes/`、`scripts/`、`docs/GODOT_MIGRATION.md`、`doc/game-design.md`、`AGENTS.md`。
- 不要在 PowerShell 用 `Get-Content | Set-Content` 处理含中文的源码或 md 文件。
- 不要把 “大招” / “施法” / “倒地” 等其它状态的精灵在本任务接入（后续任务处理）。
- 不要改变本任务规定的章节-精英映射（5 行 `[待确认]` 注释保留）。

## 【边界与限制】

- 若 `CHAPTER_LAYOUT` 不存在：B 阶段第 7 步暂用 `this.eliteTextureByChapter = {0:"enemy-elite-slime-walk", 1:"...", ...}` 在 `init` 中硬写兜底，并加注释 `// [待确认] 等地图重构后迁移至 CHAPTER_LAYOUT.eliteTexture`。
- 若用户未提供任一切好的精灵条：跳过 B 阶段全部步骤；A 阶段独立完成即视为合格。
- 若 `frameWidth=96` 与实际图片帧宽不符（运行后帧位错乱）：用 `图片宽度 / 3` 取整重算并替换，**不要** 改 `frameRate` 或 `repeat`。无法测量时控制台输出 `[待确认] <文件名> 帧尺寸未知` 并跳过该精英的接入。
- 若现有 `spawnEnemy` 实现未使用 `rankStats[rank].texture` 这种字段访问，而是其它结构：以现有结构为准，仅在 `rank === "elite"` 处条件分支替换纹理，**不要** 重构整个 `rankStats`。
- 若 `openEnemyCodex` 不存在（前置任务未做）：本任务的 A 阶段第 3 步需要顺带建立基础版（`createModalBase` + 单图） 后再扩展为分类版；若 `createModalBase` / `closeModal` / `addModalCloseButton` / `createButton` 任一基础方法缺失：停止 A 阶段，输出 `[待确认] 缺少基础 UI 方法：<方法名>`。
- 不确定 “大招”、“施法” 等状态如何接入时：本任务**不接入**，回复 `[待确认] 多状态接入需新建任务说明书`。
- 不确定章节-精英映射是否合理时：使用本说明书表格，不要自作主张交换。

## 【示例】

正例：

- 用户提供 1 张合集图 → 保存 `public/assets/enemies/elite/elite_codex.png` → A 阶段完成 → 浏览器点 “敌人图鉴 → 精英小怪” 看到 5 列原图 → `npm run build` 退出码 0。
- 用户随后提供 5 个精灵条 → B 阶段执行 → 第 3 章第 9 波出现的 elite 是 “南瓜鬼” 3 帧行走动画。

反例：

- 把合集图存到 `public/assets/enemies/elite_codex.png`（错误：路径缺 `elite/` 子目录）。
- 把 elite 的 hp 由 225 升到 280 “以匹配新美术”（错误：禁止改数值）。
- 在 spawnEnemy 里给 normal rank 也换成 “绿史莱姆精英” 纹理（错误：本任务只动 elite）。
- 把章节-精英映射调换为 “第 1 章用电猫”（错误：必须按表格）。
- 把动画 frameRate 设为 8（错误：必须 6）。
- 在 “大招” 状态下用 canvas 动态裁剪合集图制作多帧动画（错误：本任务不切片不接入多状态）。
